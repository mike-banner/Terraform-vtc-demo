<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR contributor → workflow | Une PR exécute `terraform plan` avec accès aux secrets via le runner |
| GitHub Secrets → runner env | Credentials R2 + Cloudflare exposés en variables d'environnement du job |
| Runner → backend R2 | Le runner lit/écrit le tfstate (état contenant potentiellement des valeurs sensibles) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | plan.yml — secrets dans logs | mitigate | `sensitive=true` sur variables TF + masquage GitHub Actions des secrets ; `-no-color` n'expose pas de secret |
| T-02-02 | Elevation of Privilege | apply.yml — push direct sur main | mitigate | Branch protection rule (require PR) documentée dans README (piège 5) ; trigger limité à push main/staging |
| T-02-03 | Tampering | PR malveillante déclenchant plan avec secrets | accept | Repo privé non-fork (research A1) ; checkout@v4 ne checkout pas le HEAD de fork ; plan ne mute pas l'infra |
| T-02-04 | Denial of Service | trigger sur tout push | mitigate | `paths: ['terraform/**']` limite les déclenchements (piège 2) |
| T-02-SC | Tampering | Actions GitHub tierces | accept | Toutes les actions sont officielles vérifiées (checkout, setup-terraform, github-script) — research "Audit de légitimité", pas de package npm/pip |
</threat_model>
